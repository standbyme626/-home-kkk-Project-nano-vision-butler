# Edge Model Selection Strategy (RK3566)

## 目标
- 在 RK3566（2G+16G）上为 `nano-vision-butler` 选择“可上线”的主检测模型。
- 优先保证边缘链路稳定，再追求精度上限。

## 约束前提
- 前端职责是“边缘感知与设备上报”，不是多模态主控。
- 当前链路需要主检测模型输出可稳定映射为 `bbox/class/confidence`。
- 2G 内存设备必须控制模型体量和常驻资源占用。

## 候选与排序

| 排名 | 候选 | 首发建议 | 说明 |
| --- | --- | --- | --- |
| 1 | YOLOv5n INT8 | 是 | 上线风险最低，适合作为首发基线。 |
| 2 | YOLOv8n INT8 | 是（A/B） | 精度和扩展性更有潜力，建议作为挑战组。 |
| 3 | YOLOv6n / YOLOv6Lite INT8 | 是（备选） | 偏移动端轻量路线，适合作为第三候选。 |
| 4 | YOLOv5s / YOLOv8s | 否（首发阶段） | 对 RK3566 通常过重，建议后置。 |

## 分阶段上线策略
1. 首发阶段：`YOLOv5n INT8 416`，先把主链路跑稳。
2. 提升阶段：`YOLOv8n INT8 416` 做同条件 A/B。
3. 备选阶段：`YOLOv6n INT8 416` 作为轻量方向对照。
4. 仅在收益明确时再评估 512 输入或 S 档模型。

## 决策门槛
- 先满足稳定性：连续运行无崩溃、无持续掉帧。
- 再比较质量：小目标召回和误报率优先于单次峰值 FPS。
- 最后看改造成本：预处理/后处理改造复杂度不能拖垮交付节奏。
